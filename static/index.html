#482 Add retry with backoff to the upload client
open octocat main ← feat/upload-retry
Description

Uploads currently fail hard on a transient 5xx. This adds an exponential backoff (capped at 5 attempts) around the fetch in UploadClient.send.

  • Retries on 429 / 502 / 503 / 504 and network errors.
  • Honours Retry-After when present.
  • Surfaces the final error unchanged so callers still see the real failure.
await client.send(blob, { maxRetries: 5 })
